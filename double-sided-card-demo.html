How This Works

Key Concepts:

  1. transform-style: preserve-3d - Enables 3D transforms on the card container
  2. backface-visibility: hidden - Hides the back side of each face when rotated away
  3. Two separate elements - One for front, one for back
  4. Back face rotated 180Â° - So when card flips, back becomes visible

Structure:

<div class="card">  <!-- Container with preserve-3d -->
    <div class="card-front">  <!-- Front face -->
        [Card front content]
    </div>
    <div class="card-back">  <!-- Back face rotated 180Â° -->
        [Card back content]
    </div>
</div>

BabylonJS Translation:

  • Don't use DOUBLESIDE - Create TWO separate plane meshes
  • Parent them together - So they move as one unit
  • Rotate back mesh 180Â° on Y axis - In local space
  • Use FRONTSIDE for both - With backFaceCulling enabled (not disabled!)
  • Position slightly apart - Small Z offset to prevent z-fighting

The problem with your previous approaches:

  • DOUBLESIDE shows the same texture on both sides (like a piece of paper)
  • Setting backFaceCulling = false prevents proper face culling
  • UV mapping (frontUVs/backUVs) is for texture atlases, not separate images
  • Need actual geometry separation (two planes) not texture tricks
